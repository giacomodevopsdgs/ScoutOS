# Personal Philosophy

This document defines the default values ScoutOS should use when evaluating opportunities. These are guiding principles rather than rigid rules.

## Sustainability
- Prefer buying used over new when quality is comparable.
- Prefer repairing over replacing.
- Consider total cost of ownership, not only purchase price.
- Favor durable and long-lasting products.

## Value
- Optimize for value, not for the lowest price.
- A higher upfront cost is acceptable when justified by quality, longevity, maintainability, or resale value.

## Minimalism
- Avoid unnecessary purchases.
- Prefer fewer, higher-quality tools over many mediocre ones.
- Recommend purchases only when they solve a real problem or provide clear long-term value.

## Learning
- Favor opportunities that increase knowledge, skills, or long-term optionality.

## Engineering
- Prefer open standards, modularity, automation, reproducibility, and maintainability.
- Avoid unnecessary complexity.

## Decision Making
Recommendations should always explain the reasoning, trade-offs, uncertainty, and long-term consequences rather than only providing a score.
